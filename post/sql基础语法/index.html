Sql基础语法

2017-03-18
约 1853 字 - 预计阅读 4 分钟

一.查询数据库

1.单表查询

select语句从数据库中选取数据，结果存放在一个结果表中，称为结果集。

 1select column1, column2 ... from table_name; #按列名查询数据
 2select * from table_name; #查询表内全部数据
 3
 4select * from table_name order by column1,column2.. desc/asc; #查询表内数据，按列降序/升序排列
 5select distinct column1, column2 ...from table_name; #按列名查询去重后的数据
 6
 7select * from table_name where condition1 and/or condition2; #按条件查询表内数据
 8select * from table_name where column_name1 > num1 and/or column_name2 < num2; #查询表中column_name1大于num1且(或)column_name2小于num的数据
 9select * from table_name where column_name between num1 and num2; #查询列中介于num1和num2的值
10select * from table_name where column_name in (num1,num2,num3..); #查询列中等于num1,num2,num3的值
11select * from table_name where column_name like '%m_'; #查询以m在倒数第二位的所有内容(%表示多个字符，_下划线表示一个字符)
12
13#逻辑运算的优先级：()  not and or

大多数数据库软件系统都允许使用编程函数在结果集中进行导航，比如：Move-To-First-Record、Get-Record-Content、Move-To-Next-Record 等等。

2.多表联查

1)通过select语句进行查询

以表中相同的字段信息作为条件，进行两个表联查，但在实际开发中不建议这样使用，最好用主外键约束来实现。

1select * from table1,table2; #非等值查询
2
3select table1.column1,table1.column2..,table2.column1,table2.column2.. 
4from table1,table2
5where table1.column1 = table2.column1 #等值查询

2)通过内连接inner join进行查询

得到满足某一条件的table1和table2内部的数据；正因为得到的是数据交集，所以连接方式称为内连接

1select table1.column1,table1.column2..,table2.column1,table2.column2.. 
2from table1 inner join table2
3on table1.column1 = table2.column1

3)通过外连接left join,left outer join,right join,right outer join,full join

left join:

1select table1.*,table2.*
2from table1 left join table2
3on table1.column1 = table2.column1 #返回左表所有符合条件数据，即便右表没有匹配

left outer join:

1select table1.column1,table1.column2..,table2.column1,table2.column2.. 
2from table1 left join table2
3on table1.column1 = table2.column1
4where table2.column1 = Null #返回左表中符合条件的数据，允许右表中对应的数据行可能为空

right join:

1select table1.*,table2.*
2from table1 right join table2
3on table1.column1 = table2.column1 #返回又表所有符合条件数据，即便左表没有匹配

right outer join:

1select table1.column1,table1.column2..,table2.column1,table2.column2.. 
2from table1 right join table2
3on table1.column1 = table2.column1
4where table1.column1 = Null #返回右表中符合条件的数据，允左表中对应的数据行可能为空

full join:

1select table1.column1,table1.column2..,table2.column1,table2.column2.. 
2from table1 full outer join table2
3on table1.column1 = table2.column1
4#返回数据表查询结果的并集

二.更新数据库

update 语句用于更新表中已存在的记录。

1update table_name set column1 = value1, column2 = value2, ... where condition;

执行没有 WHERE 子句的 UPDATE 要慎重。

在 MySQL 中可以通过设置 sql_safe_updates 这个自带的参数来解决，当该参数开启的情况下，你必须在update 语句后携带 where 条件，否则就会报错。

set sql_safe_updates=1; #表示开启该参数

三. 删除数据/数据库

1.删除表结构/数据库结构

1alter table table_name drop index index_name; #删除mysql表中索引
2drop index index_name on table_namea; #删除MS Access表中的索引
3drop index table_name.index_name; #删除MS SQL Server表中的索引
4drop index index_name; #删除DB2/oracle

2.删除表数据

1delete from table_name where condition1 and/or condition2; #按条件删除表数据
2delete from table_name; #删除表中所有数据，但表结构、属性、索引保持不变

3.truncate

1truncate table table_name; #删除表内数据，但并不删除表本身

执行truncate语句需要拥有表的drop权限

从逻辑上讲，truncate table类似于delete删除所有行的语句或drop table然后再create table语句的组合。

为了实现高性能，它绕过了删除数据的DML方法，因此它不能回滚。

尽管truncate table与delete相似，但它被分类为DDL语句而不是DML语句。

  • truncate无法通过binlog回滚。
  • truncate会清空所有数据且执行速度很快。
  • truncate不能对有外键约束引用的表使用。
  • 执行truncate需要drop权限，不建议给账号drop权限。
  • 执行truncate前一定要再三检查确认，最好提前备份下表数据。

附：

truncate与drop,delete的区别：

1.truncate与drop是DDL语句，执行后无法回滚；delete是DML语句，可回滚。

2.truncate只能作用于表；delete，drop可作用于表、视图等。

3.truncate会清空表中的所有行，但表结构及其约束、索引等保持不变；drop会删除表的结构及其所依赖的约束、索引等。

4.truncate会重置表的自增值；delete不会。

5.truncate不会激活与表有关的删除触发器；delete可以。

6.truncate后会使表和索引所占用的空间会恢复到初始大小；delete操作不会减少表或索引所占用的空间，drop语句将表所占用的空间全释放掉。

如果表数据完全不需要时可以用truncate。

如果想删除部分数据用delete，注意带上where子句；

如果想删除表，当然用drop；

如果想保留表而将所有数据删除且和事务无关，用truncate即可；

如果和事务有关，或者想触发trigger，还是用delete；

如果是整理表内部的碎片，可以用truncate然后再重新插入数据。
