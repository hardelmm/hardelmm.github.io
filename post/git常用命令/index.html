Git常用命令

2023-03-02
约 1015 字 - 预计阅读 3 分钟

常规流程

init本地初始化–>add更新暂存区–>commit暂存区更新至版本库–>push本地库更新至远程库

1git init
2
3git add . #添加变动
4git add -A #添加全量
5git commit -m '' #本地推送至暂存区 
6git push -u <远程库> <分支> #分支推送至远程库，并维护一条本地分支和远程分支之间的映射关系

add和commit

1git add file
2git add . #add新文件和修改文件（Git2.x版本包括删除的文件）
3git add -u #add修改和删除的文件
4git add -A #add所有文件
5
6git commit file -m ''
7git commit -am '' #am跳过add步骤

push

1git push <远程库> <本地分支>:<远程分支> #推送本地分支至远程库，与远程分支合并
2以上若远程分支名和本地分支相同，可合并简写
3git push --force <远程库> <本地分支> #忽略版本差异，强制推送
4git push <远程库> --delete <远程分支> #删除远程分支

fetch和pull

git pull 使用结果类似于git fetch + git merge ，区别在于commitID更新方式

1git fetch <远程库> <远程分支>:<本地分支> #抓取远程分支至本地
2git merge <库>/<分支> #提取数据并尝试合并到当前分支
3
4git pull <远程库> <远程分支>:<本地分支> #拉取远程库的分支至本地，与本地分支合并

reset和revert

1git reset head #取消当前版本在暂存区的缓存
2git reset --mixed <版本> #回退至指定版本 （回退重置到工作区）
3#mixed为默认参数可省略，用于重置暂存区的文件与回退版本提交(commit)保持一致，工作区文件内容保持不变
4git reset --soft <版本> #回退到指定版本（回退操作止步暂存区）
5git reset --hard <版本> #回退且删除版本间信息提交（慎用）
6
7git revert <版本> #reflog取得commit哈希值，撤销后重新push
8#revert相当于做出反向变动，push后起到对指定的commit的抵消作用

remote

remote用来操作配置远程仓库

1git remote -v 
2git remote show <仓库名>
3git remote add <新仓库> <url> #向远程主机提交新仓库
4git push -u <仓库><分支>
5
6git remote rm <仓库名> #删除远程仓库
7git remote rename <旧名称> <新名称> #修改远程仓库名

git工作流程参阅：《Git流程详解》
